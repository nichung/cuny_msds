# store URL as fragments for ease of reuse
url_a = "https://api.nytimes.com/svc/books/v3/" 
url_b = "lists/" 
url_d = "/hardcover-fiction.json?" 

books = []
book = {}

# grab values for each object returned, then store as list of dicts
for c in weeks:
    r = requests.get(url_a + url_b + c + url_d + "api-key=" + key)
    time.sleep(6) # rate limit guidance as per https://developer.nytimes.com/faq#a11
    for i in r.json()['results']['books']:
        # create dictionary for each result
        book = {
            "date": c,
            "title": i['title'], 
            "rank": i['rank'], 
            "rank_last_week": i['rank_last_week'], 
            "weeks_on_list": i['weeks_on_list'], 
            "description": i['description'], 
            "publisher": i['publisher'], 
            "author": i['author']
        }
        # append dictionary to list
        books.append(book)

# store list of dictionanies to dataframe
books_df = pd.DataFrame(books)
books_df

# save point
books_df.to_csv("books.csv")

analysis¶

summary of exploratory approach:

  • show distribution of titles relative to publishers and authors

observations:

  • a few authors over-index on the best-seller's list
  • a few publishers over-index on the best-seller's list
  • titles can be on the best-seller's list for over a year

future analysis should look at:

  • popularity of a title, author, publisher over time
  • author frequency by unique titles (which authors have most books on list?)
